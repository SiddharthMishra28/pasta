1. Rule_Category (The "Why")
MANDATORY_FIELD: Core fields that must always be populated.
CONDITIONAL_REQ: Fields that are mandatory only under specific business scenarios.
DATA_FORMAT: Ensures the string structure (Dates, IDs) matches downstream system requirements.
ALLOWED_VALUES: Validates data against a specific business domain or list.
DATA_INTEGRITY: Validates relationships between records (e.g., duplicates).
BATCH_INTEGRITY: Validates the overall payload rather than individual rows.

2. Validation_Type & Operator
Enum Name,Description,Logic Logic
NOT_NULL,Presence check,"Fails if the value is Empty, Null, or Whitespace."
MATCHES,Pattern check,Uses Regular Expressions (Regex) to validate string structure.
IN_LIST,Domain check,Compares value against a comma-separated list or JSON array.
NO_DUPLICATES,Uniqueness check,Scans the resultset for identical combinations of the target fields.
EXISTS,Metadata check,Ensures the field header/key exists in the message payload.
